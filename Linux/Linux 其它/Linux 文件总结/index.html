    开机启动相关

    /etc/issue

    • 在用户登录前显示欢迎信息
    \d      # 当日
\l      # tty 的名称
\m      # 系统的架构，uname -m
\n      # 主机名，uname -n
\o      # 域名
\r      # 内核版本
\t      # 当前时间，HH:mm:ss
\s      # 操作系统类型，Linux
\v      # 操作系统版本和构建日期，uname-v
\u      # 当前登录的用户数
\U      # 与 \u 相同，但后缀为 "user" 或 "users"（如 "1 user" 或 "10 users"）

    /etc/motd

    • 在用户登录后显示欢迎信息
    Welcome to the world of Linux

    安全

    /etc/passwd

    • 用户及其属性信息
    root:x:0:0:root:/root:/bin/bash

root        # 登录名，用户名
x           # 用户密码占位符
0           # 用户的 UID
0           # 用户主组 GID
root        # 描述信息
/root       # 用户家目录
/bin/bash   # 登录 shell

    /etc/shadow

    • 用户密码及其属性信息
    root:$6$nr...:19397:0:99999:7:7::

root        # 登录名，用户名
$6$nr...    # 加密后的密码，$6$ 中的 6 表示加密类型，!!表示没有密码
19397       # 最近一次修改密码的时间，从 1970 年 1 月 1 日起至今的天数
0           # 密码最短使用天数，即密码再过几天可以修改，0 表示随时可被变更
99999       # 密码最长使用天数，即密码再过几天必须修改，99999 表示永不过期
7           # 密码即将过期警告天数，即密码过期前几天提醒用户，默认 7
7           # 密码过期后几天帐号会被锁定
            # 账号失效时间：从 1970 年 1 月 1 日算起，多少天后帐号失效，默认为空
            # 保留字段

    /etc/group

    • 组及其属性信息
    root:x:0:admin

root    # 组名
x       # 组密码占位符
0       # 组的 GID
admin   # 以当前组为附加组的用户列表

    /etc/gshadow

    root:::
root        # 组名
            # 加密后的组密码
            # 组的管理员列表
            # 以当前组为附加组的用户

    /etc/login.defs

    • 用户 UID、组 GID、用户密码等默认配置文件

    /etc/sudoers

    /etc/sudoers 文件是 sudo 命令的主要配置文件，它包含了 sudo 命令的访问控制规则，用于控制哪些用户或用户组可以使用 sudo 命令以及这些用户或用户组可以在哪些主机上执行哪些命令。/etc/sudoers 文件是一个非常重要的系统文件，必须使用特殊的编辑器进行编辑，例如visudo命令，以确保文件的正确性和安全性。

    # User privilege specification
root    ALL=(ALL:ALL) ALL

# User alias specification
User_Alias ADMINS = user1,user2

# Cmnd alias specification
Cmnd_Alias UPDATE = /usr/bin/apt-get update
Cmnd_Alias UPGRADE = /usr/bin/apt-get upgrade

# Allow members of group sudo to execute any command
%sudo   ALL=(ALL:ALL) ALL

# Override built-in defaults
Defaults        env_reset
Defaults        secure_path="/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin"

    用户特权规范(User privilege specification)
这一部分用于定义用户的特权，包括哪些用户可以使用sudo命令以及这些用户可以在哪些主机上执行哪些命令。在上述例子中，我们定义了root用户可以在所有主机上执行所有命令，而user1用户可以在所有主机上执行/usr/bin/apt-get命令，而无需输入密码。

用户别名规范(User alias specification)：这一部分用于定义用户别名，可以方便地在用户特权规范中引用别名。在上述例子中，我们定义了一个名为ADMINS的用户别名，包括user1和user2两个用户。

命令别名规范(Cmnd alias specification)：这一部分用于定义命令别名，可以方便地在用户特权规范中引用别名。在上述例子中，我们定义了两个命令别名UPDATE和UPGRADE，分别代表/usr/bin/apt-get update和/usr/bin/apt-get upgrade命令。

用户组特权规范(Group privilege specification)：这一部分用于定义用户组的特权，可以允许用户组中的成员使用sudo命令。在上述例子中，我们允许sudo用户组的成员在所有主机上执行所有命令。

默认规范(Default specification)：这一部分用于定义sudo命令的默认选项，可以覆盖内置的默认值。在上述例子中，我们定义了env_reset选项，用于重置环境变量，以及secure_path选项，用于设置命令搜索路径。

    /etc/sudo.conf

    • sudo 命令的全局配置文件，用于设置 sudo 命令的各种全局选项和默认值。
    sudoers_file        # 指定 sudoers 文件的路径，它是 /etc/sudoers 的默认值。如果需要使用不同的 sudoers 文件，则可以使用此选项指定它的位置。
sudoers_uid         # 指定用于读取 sudoers 文件的用户 ID，默认为 0，也就是 root 用户。这是安全措施之一，可以防止其他用户修改sudoers文件。

env_reset           # 指定是否重置环境变量，使 sudo 命令的执行环境与当前用户的环境变量隔离。默认为 true，表示需要重置环境变量。
env_keep            # 指定哪些环境变量需要保留。默认为 "TERM" "COLORTERM" "DISPLAY" "XAUTHORITY" "EDITOR" "VISUAL" "SSH_AUTH_SOCK" "TZ" "LANG" "LC_*"。可以通过在文件中指定变量名称来添加或删除保留的环境变量。
secure_path         # 指定默认的安全路径，这是一个冒号分隔的路径列表，用于在执行 sudo 命令时设置 PATH 环境变量。默认值为 /usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/snap/bin。
timestamp_timeout   # 指定 sudo 命令的时间戳超时时间，默认值为 15 分钟。当用户输入一次密码后，sudo 命令会在一定时间内允许用户执行其他命令而不需要再次输入密码。可以通过修改此选项来更改默认的时间戳超时值。
passwd_timeout      # 指定在输入密码后要求用户重新输入密码的时间，以秒为单位。默认值为 5 分钟。可以通过修改此选项来更改默认的密码超时时间。
log_output          # 指定是否将命令的输出写入日志文件中。默认值为 false，表示不记录命令的输出。可以将此选项设置为 true 以记录命令输出。

    /etc/security/limits.conf

    limits.conf 是 PAM（Pluggable Authentication Modules）的一部分，用于控制用户或用户组可以打开的文件数、可以使用的内存量、可以运行的进程数等系统资源的限制。

    limits.conf 文件中的条目由四个字段组成：

    1. Domain: 指定限制适用的用户、用户组或通配符。例如，用户名、@ 开头的用户组名、* 代表所有用户，% 开头的项目代表 PAM 组。
    2. Type: 指定限制是硬限制（hard）还是软限制（soft），或者两者都适用（-）。软限制可以由用户增加到硬限制的级别，而硬限制则是用户不能超过的上限。
    3. Item: 表示要控制的资源。例如：
      • core - 核心文件的最大大小（以 KB 为单位）
      • data - 最大数据大小（以 KB 为单位）
      • fsize - 可以创建的最大文件大小（以 KB 为单位）
      • nofile - 用户可以打开的最大文件描述符数
      • stack - 最大栈大小（以 KB 为单位）
      • cpu - 最大 CPU 时间（以分钟为单位）
      • nproc - 用户可以运行的最大进程数
      • as - 地址空间限制（以 KB 为单位）
      • maxlogins - 用户可以进行的最大登录数
      • maxsyslogins - 系统范围内的最大登录数
      • priority - 进程的优先级
      • locks - 用户可以持有的最大文件锁数
      • sigpending - 用户可以排队的最大信号数
      • msgqueue - POSIX 消息队列的最大字节数
      • nice - 用户可以设置的最大 nice 值
      • rtprio - 用户可以设置的最大实时优先级
    4. Value: 为上述条目设置的限制值，具体取决于条目类型。
    *       soft    nofile      1024
*       hard    nofile      65535
@group  hard    nproc       15000
root    hard    as          1000000
root    soft    nproc       1300

    硬件相关

    /proc/cpuinfo

    输出项 含义
    processor 系统中逻辑处理核的编号。对于单核处理器，则可认为是其CPU编号，对于多核处理器则可以是物理核、或者使用超线程技术虚拟的逻辑核
    vendor_id CPU制造商
    cpu family CPU产品系列代号
    model CPU属于其系列中的哪一代的代号
    model name CPU属于的名字及其编号、标称主频
    stepping CPU属于制作更新版本
    cpu MHz CPU的实际使用主频
    cache size CPU二级缓存大小
    physical id 单个CPU的标号
    siblings 单个CPU逻辑物理核数
    core id 当前物理核在其所处CPU中的编号，这个编号不一定连续
    cpu cores 该逻辑核所处CPU的物理核数
    apicid 用来区分不同逻辑核的编号，系统中每个逻辑核的此编号必然不同，此编号不一定连续
    fpu 是否具有浮点运算单元（Floating Point Unit）
    fpu_exception 是否支持浮点计算异常
    cpuid level 执行cpuid指令前，eax寄存器中的值，根据不同的值cpuid指令会返回不同的内容
    wp 表明当前CPU是否在内核态支持对用户空间的写保护（Write Protection）
    flags 当前CPU支持的功能
    bogomips 在系统内核启动时粗略测算的CPU速度（Million Instructions Per Second）
    clflush size 每次刷新缓存的大小单位
    cache_alignment 缓存地址对齐单位
    address sizes 可访问地址空间位数
